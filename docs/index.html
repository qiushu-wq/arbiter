The Operating System between Agents and LLMs

Arbiter 不是 Agent 框架。它是 Agent 与 LLM 之间的运行时 —— 管理上下文窗口分配、工具调用调度、状态持久化与故障恢复。就像操作系统管理 CPU 和内存一样，Arbiter 管理 token budget 和 tool invocation。

Architecture

三层 Ring 架构

Arbiter 采用分层防御设计。Ring 1 是核心运行时保证，Ring 2 是自适应中间件，Ring 3 是管理平面。每一层独立可替换，不锁厂商。

Ring 1 · Foundation

核心运行时

最内层保证。固定分区配额、即时回收、12字段审计、异常护栏。不依赖任何外部服务。

Core Audit Guard Status
Ring 2 · Adaptive

自适应中间件

基于运行数据动态调整：自动愈合、配额学习、硬限制熔断、优雅关闭。Ring 1 不变的前提下扩展能力。

Heal Adapt Cap Drain Lock
Ring 3 · Control Plane

管理平面

可视化仪表盘、配额策略管理、审计轨迹查询、企业级 RBAC。开发者通过 Studio 面板观察和调优运行时。

Studio Enterprise RBAC API

Install

一行命令，即刻开始

pip install 从 GitHub 直接安装。同时获得 arbiter-doctor 诊断工具 + arbiter-lite 配额管理库。

pip (推荐)
$ pip install git+https://github.com/qiushu-wq/arbiter.git
一行命令，arbiter-doctor + arbiter-lite 全装上。Python 3.9+，零依赖。
诊断项目
$ arbiter-doctor ./your-project
# 或直接跑 Python
$ python arbiter_doctor.py ./your-project
扫描项目目录，找出上下文共享、故障盲区、状态冲突三类潜在问题。
代码集成
$ python -c "from arbiter_lite import QuotaManager; print('OK')"
40 行配额管理器。固定分区 + 闲置回收，直接 import 即用。
